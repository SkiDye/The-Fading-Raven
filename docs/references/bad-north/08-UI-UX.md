# Bad North - UI/UX 시스템

## 디자인 철학

- **미니멀리즘**: 불필요한 UI 요소 제거
- **직관적 피드백**: 시각적 요소로 정보 전달
- **컨텍스트 기반**: 상호작용 시에만 관련 정보 표시
- 귀여운 비주얼 vs 잔혹한 전투의 대비

---

## 메인 메뉴

### 구성요소
- **타이틀**: "BAD NORTH" 로고 (상단 중앙)
- **메뉴 옵션** (세로 중앙 정렬):
  - Continue (계속하기)
  - New Game (새 게임)
  - Settings (설정)
  - Exit (나가기)
- **배경**: 동적 섬 풍경 (실제 게임플레이 화면)

### 디자인 특징
- 흰색 텍스트, 반투명 배경 없음
- 선택된 항목은 밝은 하이라이트
- 산세리프 폰트, 가독성 높음

## 인게임 HUD

### 기본 구성 ✅

| 요소 | 위치 | 설명 |
|------|------|------|
| 커맨더 초상화 | 좌하단 | 원형, 색상 배경 |
| 커맨더 이름 | 초상화 아래 | 예: "Idelisa" |
| 명령 버튼 3개 | 초상화 우측 | 깃발/십자/핀 아이콘 |
| 일시정지 | 우상단 | **\|\|** 아이콘 |
| 골드 표시 | 우상단 | 코인 아이콘 + 숫자 |

### 명령 버튼 분석 ✅

| 아이콘 | 기능 | 비고 |
|--------|------|------|
| 깃발 + 배 | **Flee (도주)** | 왼쪽 버튼 |
| 십자가 + 집 | **Replenish (회복)** | 중앙 버튼 |
| 핀 마커 | **Move (이동)** | 오른쪽, 노란색 활성화 |

### 유닛 선택 피드백 ✅

| 상태 | 시각 표현 |
|------|----------|
| 선택됨 | 유닛 위 **녹색 원형 아이콘** |
| 이동 가능 | 타일에 **흰색 핀 마커** |
| 비선택 | 기본 상태 (표시 없음) |

> **참고**: 갓레이(God Ray) 효과 **없음** - 깔끔한 원형 선택 표시 사용

## 웨이브 알림

### 구성요소
- **방향 화살표**: 적 침공 방향을 화면 가장자리에 표시
- **보트 아이콘**: 접근하는 바이킹 보트 직접 보임
- **Final Wave 텍스트**: 마지막 웨이브 시 화면 중앙 표시

### 디자인 특징
- 화면 밖 침공 시 화살표 UI로 방향 표시
- 화살표 색상/크기로 위협 수준 암시
- 텍스트 알림 최소화 - 시각적 요소로 전달

## 월드맵 UI ✅

### 구성요소
- **섬 노드**: 각 섬을 점/아이콘으로 표시
- **연결선**: 섬 간 이동 경로
- **보상 아이콘**: 획득 가능 보상 미리보기
- **난이도 표시**: 적 숫자/아이콘으로 암시

### 노드 상태

| 상태 | 시각 표현 |
|------|----------|
| 클리어/점령 | 어둡거나 빨간색 |
| 선택 가능 | 밝게 하이라이트 |
| 현재 선택 | **노란색** |

### 관측 데이터

| 요소 | 값 |
|------|-----|
| 커맨더 표시 | "4/4 Commanders Available" |
| 턴 버튼 | "Next Turn" + **Y키** |
| 일시정지 | 우상단 **\|\|** |
| 섬 이름 | Stack, Coll, Tower Peak 등 |

## UI/UX 디자인 원칙 요약

### 1. 미니멀리즘
- 불필요한 UI 요소 제거
- 게임플레이에 집중할 수 있는 깔끔한 화면
- 정보는 필요할 때만 표시

### 2. 직관적 피드백
- 선택 = 하이라이트 (원형 링, 밝기 변화)
- 위협 = 방향 화살표 + 시각적 요소
- 진행 = 간단한 노드 맵

### 3. 일관된 비주얼 언어
- 전체적으로 통일된 색상 팔레트
- 산세리프 폰트
- 반투명 요소 최소화

### 4. 로그라이트 특성 반영
- 퍼마데스를 명확히 표시
- 진행 상황을 시각적으로 전달
- 선택의 무게감을 UI로 표현
